java安全1-反射与RCE、SQL注入等漏洞基础

2025年06月20日

java安全1-反射与RCE、SQL注入等漏洞基础

ClassLoader（类加载机制）

java依附于JVM-java虚拟机，运行前，先将进程编译为class文件，通过java.lang.classloader加载类字节码，类加载机制会调用JVM的方法进行实例定义

类动态加载方法

其中反射会默认初始化被加载的静态属性与方法，所以可以由此写入任意代码

BCEL加载类

BCEL全名Apache Commons BCEL，其中om.sun.org.apache.bcel.internal.util.ClassLoader加载类，会将字符串转为字节码，再将解析后的字节码注册，一旦该类被加载就会触发字节码中的恶意代码。因此BCEL被应用于各种payload。一旦包含了$$BCEL$$关键字，BCEL就会使用特殊的方式进行解码并加载解码之后的类

用BCEL进行编码

Xalan加载类

Xalan中的com.sun.org.apache.xalan.internal.xsltc.trax.TemplatesImpl也可传入类字节码，并且初始化，从而实现RCE与反序列化

反射

反射机制可以实现Java类的动态调用。

反射java.lang.Runtime

runtime中有一个exec方法可进行命令，常调用其进行命令执行

由于Runtime的构造是private方法，不能够直接调用其构造方法

反射创建类实例

public类Runtiome不希望除自身外任何人创建该实例，通过反射创建

调用类方法

提供了一个获取某个类的所有的成员方法的方法

调用成员变量

可获取所有成员变量名称，也可不论权限修改变量值

sun.misc.Unsafe

JAVA底层API提供的一个对底层内存、CAS、类对象等操作的不安全类，为内部API，由final字样定义，同样由反射调用

也可以用field反射调用成员变量，还存在一个无视一切的方法allocateInstance，使用Unsafe创建UnSafeTest类实例

文件系统安全

文件系统对象为java.io.FileSystem，且分为io与nio两个文件系统

其中三个类：FileInputStream（读取文件）、FileOutputStream（写文件）、RandomAccessFile（读写文件）

FileSystemProvider是NIO2新增读取方法，基于NIO的文件读取逻辑，打开FileChannel，读取Channel内容

文件名空字节截断漏洞

JAVA jdk更新后，java.io.File类中多了一个isInvalid方法，用来检测文件名为空的情况

这个漏洞多用于文件上传，用来截断文件后缀，例如payload：

起到一个截断的作用，也可以用双扩展

命令执行

其中runtime等类在JDK 自带的核心类库（rt.jar / modules），审计代码时多关注Runtime.exec/ProcessBuilder/ProcessImpl等关键词

Runtime命令执行

java.lang.Runtime类中由exec函数，可以进行命令执行，一般反射调用

ProcessBuilder命令执行

Runtime的exec会调用ProcessBuilder来执行命令，

注意Runtime和ProcessBuilder都不是程序的最终执行点

UNIXProcess/ProcessImpl

最终调用native方法（JNI也就是java native interface）的forkAndExec命令执行的类，通过fork&exec进行命令执行

forkAndExec命令执行-Unsafe+反射+Native方法调用

同样通过反射以及Unsafe特性：

使用sun.misc.Unsafe.allocateInstance(Class)特性可以无需new或者newInstance创建UNIXProcess/ProcessImpl类对象；反射UNIXProcess/ProcessImpl类的forkAndExec方法；构造forkAndExec需要的参数并调用；反射UNIXProcess/ProcessImpl类的initStreams方法初始化输入输出结果流对象；反射UNIXProcess/ProcessImpl类的getInputStream方法获取本地命令执行结果(如果要输出流、异常流反射对应方法即可)

JDBC

JDBC是Java与数据库连接操作的API

java通过java.sql.DriverManger来管理数据库驱动，现在java.sql.DriverManger注册驱动，再通过getConnection连接

DataSource连接

直接用DataSource对象连接数据库，在xml或properties中配置即可

Springboot中在application.properties或application.yml中定义

Tomcat JNDI DataSource与Resin JNDI DataSource也可获取（JNDI(Java Naming and Directory Interface)）
